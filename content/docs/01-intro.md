## 一个电化学工程师，为什么敢碰全栈开发？

我是一名做电池电化学仿真的工程师，日常工作是 COMSOL 建模和电池智能设计与优化、寿命预测算法。Python 和 scikit-learn 我比较熟悉，但 **Django 是什么？HTML 怎么写？前端和后端怎么通信？Celery 怎么配？** 这些我统统不知道。

但我有一个明确的需求：**我想做一个网页平台，让我和团队可以上传实验数据，自动跑机器学习模型，保存训练结果，还能在线预测。** 不需要找前端，不需要找后端，就我一个人，加上 AI。

这就是 **Vibe Coding**——你不需要会所有的技术，你需要的是：

1. **清楚地知道自己要什么**
2. **会用自然语言把需求拆解给 AI**
3. **能判断 AI 给出的方案是否合理**

> [!NOTE]
> **这篇文章的重点不是代码**，而是我和 AI 之间的每一轮对话是如何设计的——提示词怎么写、AI 建议了什么、我做了哪些调整、踩了哪些坑。

## ML 平台概览

我要做的平台叫 **ML计算平台**，核心工作流 7 步：

```
用户登录 → 上传数据 → 选择特征和标签 → 选择学习类型 → 训练+调参 → 保存模型 → 在线预测
```

### 最终技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Bootstrap 3.4.1 + jQuery 3.6.0 + ECharts 5.x |
| 后端 | Django 3.2.9（Template + JsonResponse API） |
| 异步 | Celery + SQLite broker（Windows solo pool） |
| 数据库 | MySQL 5.7.31 |
| ML | scikit-learn + lazypredict + Optuna + pandas + numpy |



### 开发周期

我从零开始搭建这个平台用了大约 **4 天**（不包含前期 2 周学习 Django/MySQL/JS 基础的时间）。如果纯靠自己，这个时间可能是 2-3 个月，而且大概率中途放弃。
